Yazılım Testi

Yazılım Testinin Amacı

Yazılım testinin temel amacı, yazılımın hatasız çalışmasını sağlamak, yazılımda var olabilecek kusurları bulmak ve kullanıcı beklentilerine uygun şekilde çalıştığından emin olmaktır.

Test Türleri

  • Fonksiyonel Testler: Yazılımın işlevselliğini doğrulamak.
  • Fonksiyonel Olmayan Testler: Yazılımın performansı, güvenliği ve diğer dışsal faktörlere dayanıklılığını test eder.

Test Düzeyleri

  • Unit Test (Birim Testi): Yazılımın her bir biriminin (fonksiyon, metot vb.) doğru çalışıp çalışmadığını test eder.
  • Integration Test (Bütünleştirme Testi): Yazılımın farklı bileşenlerinin bir arada nasıl çalıştığını test eder.
  • System Test (Sistem Testi): Yazılımın tüm bileşenlerinin bir araya getirilip, sistem olarak düzgün çalışıp çalışmadığını test eder.
  • Acceptance Test (Kabul Testi): Yazılımın müşteri gereksinimlerine uygun olup olmadığını test eder.

Test Yöntemleri

  • Siyah Kutu Testi: Yazılımın işlevselliğine odaklanır ve iç yapıyı bilmez.
  • Beyaz Kutu Testi: Yazılımın iç yapısını bilerek yapılan testtir.
  • Gray Box Testi: Hem iç yapıyı hem de işlevselliği test eder.

Testin Otomasyonu

Test otomasyonu, testlerin manuel olarak yapılmak yerine otomatikleştirilmesini sağlar. Bu, büyük projelerde zaman ve kaynak tasarrufu sağlar.

Test Yaşam Döngüsü (STLC)

  • Test Planlaması: Test stratejisinin belirlenmesi.
  • Test Tasarımı: Test senaryolarının oluşturulması.
  • Test Uygulaması: Testlerin gerçekleştirilmesi.
  • Hata Raporlama: Test sırasında bulunan hataların raporlanması.
  • Test Kapanışı: Test sürecinin sonuçlarının değerlendirilmesi.
